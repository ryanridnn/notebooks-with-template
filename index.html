Example 1. Hello, World!¶

We record a function and run the forward and reverse AADC kernels.

Recording and the forward kernel¶

We record a simple function $f=\exp{\left(\frac{x}{y}+z\right)}$ and check that the forward AADC kernel gives the same result as the original function.

First let's include necessary headers

In [1]:
#include <iostream>
                            #include <aadc/aadc.h>

We can declare variables as idouble. Note that you can use these same way as native double type

In [2]:
idouble x(0.5),y(1.1),z,f;

Note that size of idouble is the same as native double

In [3]:
assert(sizeof(double) == sizeof(idouble));
              std::cout << " double size : " << sizeof(double) << " bytes"
                  << "\nidouble size : " << sizeof(idouble) << " bytes";
              
 double size : 8 bytes 
idouble size : 8 bytes

And internally idouble simply hold a value and nothing else

In [4]:
std::cout << x.val;
              
0.5

Normally AAD libraries hold variable id or a pointer to internal structure, so you may ask how we track variables. The answer is "we use variable address!". This has multiple advantages like identical memory layout for active and native objects. I.e. std::vector<double> and std::vector<idouble> have same memory representations!

As you expect from native double type, you can use idouble types in expressions

In [5]:
z = x + sin(y);
              std::cout << "z:" << z;
              
z:1.39121

This means integration into existing code base is very easy. Just change native double type to active double type, recompile and your code should produce the same results!

Recording calculations¶

Now that we have active types in, we can begin recording calculations. We start by selecting vectorization type we want to use (avx2 or avx512).

In [6]:
typedef __m256d mmType; // __m512d is used for AVX512

We next create aad_funcs object that going to hold recorded calculations for forward kernel and adjoint kernel.

In [7]:
aadc::AADCFunctions<mmType> aad_funcs;

Next we call .startRecording() to initialize AADC library and start recording of forward and adjoint kernels.

In [8]:
aad_funcs.startRecording();
              
You are using evaluation version of AADC. Expire date is 20230201

We should select which variables are inputs to valuation graph and keep argument handle.

In [10]:
aadc::AADCArgument x_arg(x.markAsInput());
                  aadc::AADCArgument y_arg(y.markAsInput());
                  aadc::AADCArgument z_arg(z.markAsInput());
